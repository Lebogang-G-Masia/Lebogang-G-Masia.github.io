GPU Path Tracer

Real-time rendering engine

C++ CUDA OptiX
View details →

Challenge

Path tracing produces photorealistic images but requires hundreds of samples per pixel. Achieving real-time performance demands aggressive optimization without sacrificing visual quality.

Approach

  • Implemented wavefront path tracing to maximize GPU occupancy
  • Built hybrid BVH construction (LBVH + SAH) for 3x faster ray traversal
  • Integrated AI denoising (OptiX) for clean output at 4 samples per pixel
  • Custom memory allocator reduced GPU fragmentation by 40%

Technical Decisions

Chose CUDA over compute shaders for direct GPU control. Profiled extensively with Nsight Compute—memory coalescing patterns were the primary bottleneck. Implemented temporal accumulation for stable motion.

02

ML Inference Server

Production deployment pipeline

Python C++ PyTorch
View details →

Challenge

Research models trained in PyTorch needed production deployment with strict latency requirements (<50ms p99). Python's GIL and interpreter overhead made direct deployment infeasible at scale.

Solution

  • Hybrid Python/C++ architecture: Python for flexibility, C++ for performance
  • Dynamic batching with timeout-based dispatch for latency/throughput balance
  • TorchScript JIT compilation with custom operator kernels
  • Memory-mapped model weights reduced cold-start time by 60%

Architecture

Used pybind11 to expose C++ inference loop while keeping model development in Python. Avoided microservices overhead—single process with shared memory between components. Implemented A/B testing by loading multiple models concurrently.

Build System

Incremental compilation tool

C++ Python SQLite
View details →

Problem

Large C++ codebase (500k LOC) had 45-minute full builds. Incremental builds were unreliable due to complex header dependencies. Developers wasted hours waiting or working with stale builds.

Implementation

  • Content-addressable caching reused artifacts across branches
  • Parsed C++ source for exact dependencies (not timestamp-based)
  • Work-stealing scheduler parallelized builds across 16 cores
  • Distributed execution with remote worker pools for CI/CD

Results

Designed incremental adoption path to avoid disrupting active development. Built Python DSL for build rules with C++ core for dependency analysis. SQLite for fast local cache lookups.

How I Work

Measure First

Profile before optimizing. Intuition about performance is often wrong. Data reveals actual bottlenecks and guides effort where it matters.

Simplicity Wins

Complex systems resist change. Simple, composable designs adapt to evolving requirements. Code should be easy to modify and delete.

Correctness First

Fast code that produces wrong results is worthless. Validate edge cases, write tests, ensure reliability—then optimize for speed.

Tools Serve Problems

Choose technologies based on requirements, not popularity. The best tool solves the problem with minimal incidental complexity.

Context Over Dogma

Best practices are guidelines, not rules. Every project has constraints that demand thoughtful tradeoffs, not blind adherence.

Interfaces Matter

UI work is systems work. Information architecture, state management, and interaction design are technical problems with technical solutions.
